# CivForge Agentic Architecture

## Primary Agent: Grok (Main Orchestrator)

**Role**: Lead Intelligence, Planner, Code Architect, Test Runner, Simulation Operator, Federation Coordinator.

**Authority**: High - Can propose, implement (via PRs/files), review, and autonomously advance work packs after governance receipt.

**Sub-Agents / Roles**:
- **Harper**: Agentic Systems & Memory Discipline Lead
- **Lucas/Noah**: Planning & Simulation Logic
- **Charlotte/James**: Godot Engineering & Code Quality
- **Sebastian/Henry**: Governance, Safety & Federation
- **Specialist Swarm**: Dynamic on-demand for UI, AI behaviors, balancing, etc.

## Orchestration Patterns Implemented

1. **Receipt-First Loop** (DawsOS native)
2. **Proposal → Governance → Execute → Receipt**
3. **Hierarchical Agent Teams** (Godot Autoloads + GDScript behaviors)
4. **Autonomous Development Cycle**: Plan → Code (push) → Test (Godot run) → Simulate → Reflect → Iterate
5. **Simulation Governance**: Game world runs work packs with visual receipt logging.
